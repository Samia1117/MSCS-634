# MSCS-634 Lab 1: Data Visualization, Preprocessing, and Statistical Analysis

## Purpose

This lab applies core data mining prep steps to a retail sales dataset: loading data into
pandas, visualizing it with several chart types, cleaning it up (missing values, outliers,
reduction, scaling/discretization), and computing descriptive statistics and correlations.
The goal is to practice the full pipeline that comes before any modeling step - understanding
and preparing raw data.

## Dataset

`data/retail_sales.csv` is a synthetic retail sales dataset (1,000 orders) generated by
`generate_dataset.py`. It's modeled on typical e-commerce order data - date, region, product
category, units sold, unit price, discount, customer age, and revenue. Missing values and a
handful of outliers were injected on purpose so the preprocessing steps in the notebook have
real issues to detect and fix, rather than working with an already-clean dataset.

## Key insights

- Revenue is driven more by unit price than by units sold - `Unit_Price` correlates with
  `Sales_Revenue` at about 0.76, while `Units_Sold` only correlates at about 0.30. Electronics,
  which has the highest price point, brings in noticeably more total revenue than other
  categories even though order volume is similar across categories.
- Orders are split roughly evenly across the four regions (~25% each), which reflects how the
  data was generated rather than a real regional pattern.
- The IQR method on `Sales_Revenue` flagged about 12% of orders as outliers - this includes the
  large bulk orders (150+ units) that were injected on purpose, but also a sizeable chunk of
  naturally high-revenue orders, since `Sales_Revenue` turned out to be right-skewed. This was
  a good reminder that IQR fences react to the shape of the distribution, not just planted
  anomalies, so the flagged set is worth a sanity check rather than removing blindly.
- `Customer_Age` doesn't correlate meaningfully with any other variable, which makes sense
  since it was generated independently of purchase behavior - a reminder that correlation
  analysis is also useful for confirming when variables are *not* related.

## Challenges and decisions

- **Choosing a dataset:** Rather than pull a dataset from Kaggle/Data.gov, I generated a
  synthetic retail sales dataset with `numpy`/`pandas`. This made it possible to control exactly
  where missing values and outliers appear, so every preprocessing step in the lab
  (missing-value handling, IQR outlier detection) has something real to work with instead of
  hoping a downloaded dataset happens to have the right issues.
- **Imputation choices:** Missing values were filled rather than dropped, since the missing rate
  per column was low (2-5%) - dropping rows would have thrown away otherwise-valid data. Mode
  was used for the categorical `Region` column, mean for `Discount_Percent`, and median for
  `Customer_Age` to be a bit more robust to any skew.
- **Outlier handling:** Outliers were detected using the IQR method on `Sales_Revenue` rather
  than standard deviation, since IQR is less sensitive to the outliers themselves when setting
  the threshold.
