# MSCS-634 Lab 2: Classification Using KNN and RNN Algorithms

## Purpose

This lab compares two distance-based classifiers - K-Nearest Neighbors (KNN) and Radius
Neighbors (RNN) - on the Wine dataset from `sklearn.datasets`. The goal is to see how each
model's accuracy changes as its main hyperparameter changes (number of neighbors for KNN,
radius for RNN), and to understand the tradeoffs between a fixed-neighbor-count approach and a
fixed-distance approach to classification.

## Dataset

The Wine dataset (built into scikit-learn) has 178 samples, 13 numeric chemical-property
features (alcohol, malic acid, ash, magnesium, flavanoids, proline, etc.), and 3 target classes
representing different wine cultivars (59 / 71 / 48 samples respectively). The features are used
on their original, unscaled measurement ranges rather than being normalized, since the RNN
radius values specified in the lab instructions (350-600) only make sense at that scale -
`proline` alone ranges from about 278 to 1680.

## Key insights

- **KNN accuracy (k = 1, 5, 11, 15, 21):** 0.778, 0.806, 0.806, 0.806, 0.806. Accuracy jumps
  from k=1 to k=5 and then plateaus - a single neighbor is noisy and easily swayed by one
  mislabeled or borderline point, but once enough neighbors vote together the estimate
  stabilizes. Increasing k further (up to 21) doesn't hurt, since the three wine classes are
  reasonably well separated in feature space.
- **RNN accuracy (radius = 350, 400, 450, 500, 550, 600):** 0.722, 0.694, 0.694, 0.694, 0.667,
  0.667. Accuracy trends downward as radius grows - a small radius keeps the neighborhood tight
  and mostly single-class, while a larger radius starts pulling in points from neighboring
  classes and dilutes the majority vote.
- **KNN outperformed RNN at every setting tested** (best KNN ~0.81 vs. best RNN ~0.72). KNN's
  fixed neighbor count adapts automatically to how densely points are packed in different parts
  of feature space, while RNN's fixed radius has to be hand-tuned to the data's scale and
  density - too small risks sparse or empty neighborhoods, too large mixes in the wrong class.
- RNN is still a reasonable choice when "close" has a real physical meaning (a fixed distance
  threshold matters more than a fixed neighbor count), but for a dataset like Wine, with
  features on very different scales and no obvious natural radius, KNN was the more reliable
  and easier to tune classifier.

## Challenges and decisions

- **Feature scaling:** The lab instructions specify RNN radius values of 350-600, which only
  produce meaningful (non-degenerate) neighborhoods if the features are left unscaled - on
  standardized data, distances that large would be effectively meaningless (nearly every point
  would be a "neighbor"). This was confirmed by checking the feature ranges directly (`proline`
  alone spans roughly 278-1680), so the notebook intentionally does not apply Min-Max or
  Z-score scaling before fitting either classifier.
- **Stratified split:** The 80/20 train/test split uses `stratify=y` so all three wine classes
  keep roughly the same proportions in both sets, given the dataset is only moderately balanced
  (59/71/48 samples per class).
- **RNN edge cases:** With the given radius values, `RadiusNeighborsClassifier`'s default
  settings worked without any test point falling outside every training point's radius (which
  would otherwise raise an error or require an `outlier_label`). This was verified before
  finalizing the notebook.
